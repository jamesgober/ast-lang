# ast-lang v1.0.0 — API freeze

**Stable.** v1.0.0 freezes the public API. The node trait and the visitor /
transform machinery built in 0.2.0 are now under a Semantic Versioning promise: no
breaking change before `2.0`, additions only in minor releases, MSRV (Rust 1.85)
rising only in a minor. There are no functional API changes from `0.2.0`; this
release is the commitment.

## What is ast-lang?

The syntax-tree substrate of the `-lang` family: the trait an AST node implements
and the generic machinery that walks and rewrites a tree of those nodes. It owns no
grammar — a language brings its own node type — so one set of traversal code serves
every language built on it. Nodes live in an [`arena-lang`] `Arena`, wired by `Id`
handles; ast-lang adds the [`Node`] contract and the traversals over it.

## The frozen surface

- **`Node`** — `span`, `each_child`, `map_children`: the three operations a
  language's node type implements.
- **`Visitor`** + **`Flow`** — a read-only traversal (`enter`/`leave`) and the
  control enum that steers it (`Continue` / `SkipChildren` / `Stop`).
- **`walk`** — iterative, depth-first read traversal; stack-safe on any depth.
- **`transform`** — iterative, post-order tree rebuild into a destination arena.
- **Re-exports** — `Arena`, `Id` (arena-lang) and `Span` (span-lang).

The full reference, the SemVer promise, and what is deliberately left out (a
`serde` feature, a bundled tree type, mutating/fallible visitors — all addable
later without a breaking change) are in
[`docs/API.md`](https://github.com/jamesgober/ast-lang/blob/main/docs/API.md).

## Guarantees

- **Total traversal.** A walk visits every node of a tree exactly once; pre-order
  enters a parent before its children, post-order leaves children before the
  parent; children are visited in source order. Held by property tests against the
  tree structure.
- **Stack-safe.** Both `walk` and `transform` are iterative — a 200,000-node deep
  chain is traversed and rebuilt without recursing on the call stack.
- **Faithful rewriting.** An identity `transform` reproduces the tree's shape and
  the span on every node.
- **No panics.** A handle that names no live node, or transforming from an
  out-of-range root, is a defined `Option`/no-op outcome.
- **`no_std`** with `alloc`; `#![forbid(unsafe_code)]`; no `unwrap`/`expect`/`todo!`
  in shipping code.

## Breaking changes

**None.** Identical API to `0.2.0`. From here, breaking changes require `2.0`.
