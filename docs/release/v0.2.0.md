# ast-lang v0.2.0 — Node trait & traversal machinery

**The core, front-loaded.** v0.2.0 turns the scaffold into a working syntax-tree
substrate: the [`Node`] trait a language's node type implements, and the generic
visitor and transform machinery that walk and rewrite a tree of those nodes. This
was the hard part of the roadmap, and the roadmap put it first.

## What is ast-lang?

The syntax-tree layer of the `-lang` family. It owns no grammar: a language defines
its own node type, stores nodes in an [`arena-lang`] `Arena`, and wires the tree
with `Id` handles; ast-lang supplies the trait those nodes satisfy and the
language-agnostic traversals over them. It is the layer a parser builds into and
every later pass walks.

## What's new in 0.2.0

### `Node` — three operations, no grammar

A language's node type implements [`Node`] to report its span, enumerate its direct
child handles, and rebuild itself with remapped children:

```rust
use ast_lang::{Id, Node, Span};

enum Expr {
    Lit(i64, Span),
    Add(Id<Expr>, Id<Expr>, Span),
}

impl Node for Expr {
    fn span(&self) -> Span {
        match self { Expr::Lit(_, s) | Expr::Add(_, _, s) => *s }
    }
    fn each_child(&self, f: &mut dyn FnMut(Id<Self>)) {
        if let Expr::Add(a, b, _) = self { f(*a); f(*b); }
    }
    fn map_children(&self, f: &mut dyn FnMut(Id<Self>) -> Id<Self>) -> Self {
        match self {
            Expr::Lit(v, s) => Expr::Lit(*v, *s),
            Expr::Add(a, b, s) => Expr::Add(f(*a), f(*b), *s),
        }
    }
}
```

Read traversal is built on `each_child`; rewriting is built on `map_children`. A
leaf is an empty `each_child` and a plain copy.

### `walk` — iterative, stack-safe read traversal

[`walk`] drives a [`Visitor`] (`enter`/`leave`) over the tree depth-first. It keeps
its work stack on the heap rather than recursing, so a pathologically deep tree —
the test suite walks a 200,000-node chain — is traversed without overflowing the
call stack. The visitor steers the walk with a [`Flow`]: continue into children,
skip them, or stop the whole traversal.

### `transform` — iterative tree rewriting

[`transform`] rebuilds a tree into a destination arena, passing each node through a
closure post-order, so a node's children are already rebuilt and remapped when the
closure sees it. Identity yields a faithful deep copy — same shape, same spans.
Like `walk`, it is iterative and never panics: an out-of-range root or a full
destination arena returns `None`.

### Tests and benchmarks

`tests/traversal.rs` holds exact enter/leave order, pruning, stopping, the
out-of-range no-op, identity copy, and the deep-tree case byte for byte;
`tests/properties.rs` cross-checks the order invariants — every node visited once,
parents bracketing their children, source-order children — against the tree's own
structure over a wide `proptest` space. Benchmarks measure `walk` and `transform`
as the tree grows.
